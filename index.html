Background

Once upon a time there is a developer who made a ton of site for clients and keeps on maintaining clients website, eventually this became a problem as more clients come, more sites are being consistently being maintained, the developer came up the idea of creating scripts for testing applications instead of redundantly testing them each time, now there's no worry on checking sites and finding hidden little errors wherever they hide.

How can a developer overcome overwhelming demands and to make sure everything is working? — As a developer, you are responsible to make sure your work is actually working by any means, making a script is the most logical way to solve this.

Why not make a selenium server to test everything for you? — This is ideal if your coding directly in production, as time occurs different environments are created and selenium won't cover that — Phpunit has a special function that tests your app without running the server, this is useful if you're going to move from environment to environment.

Why not just test each micro services? — There are testing for generality of the application and parts of the application, in this tutorial we are going to cover testing generality of the application. — How do you usually check a site? Usually when I am checking an application you have to check the bigger picture first, then focus on the little things after.

It is recommended to install phpunit within the repository's vendor, since we need our vendor dependencies included to run the tests.

You must have composer installed from your git repository in order to run this, go to the root of your git repository and execute this command

<?php

use Zend\Test\PHPUnit\Controller\AbstractHttpControllerTestCase;

class UnitTest extends AbstractHttpControllerTestCase
{
    public function firstStepTest()
    {
        $firstVariable = 1;        
        $secondVariable = 1;
        
        $sum = $firstVariable + $secondVariable;

        // Let's check if 1 + 1 is in fact 2
        $this->assertTrue($sum == 2);
    }
}

?>

<?php

use Zend\Test\PHPUnit\Controller\AbstractHttpControllerTestCase;

class UnitTest extends AbstractHttpControllerTestCase
{
    public function setUp()
    {
        $this->setApplicationConfig(
            include getcwd() .'/config/application.config.php'
        );
        parent::setUp();
    }

    protected function getConfig()
    {
        return $this->getServiceManager()->get('config');
    }

    public function verifyingConfigTest()
    {
        $config = $this->getConfig();

        // Let's verify if we are getting configuration files
        $this->assertTrue(!empty($config));

        // Let's verify again that we are getting included files like doctrine        
        $this->assertTrue(isset($config['doctrine']));
    }
}

?>

Congratulations, you now have demonstrated on how to verify variables from services of your running ZendFramework 2

<?php

use Zend\Test\PHPUnit\Controller\AbstractHttpControllerTestCase;

class UnitTest extends AbstractHttpControllerTestCase
{
    public function setUp()
    {
        $this->setApplicationConfig(
            include getcwd() .'/config/application.config.php'
        );
        parent::setUp();
    }

    protected function getHtml()
    {
        return $this->getResponse()->getContent();
    }

    public function visitAppsIndexPageTest()
    {
        $this->dispatch('/');

        // Let's Verify that we recieved a HTTP 200 OK status
        $this->assertResponseStatusCode(200);

        // Let's check the HTML recieved
        $html = $this->getHtml();

        // Now here's where RegEx comes in
        $pattern = '/Hello World!/';

        // Let's Verify that we are getting the right content for that page.
        $this->assertTrue(preg_match($pattern, $html) != 0);
    }
}

?>

Congratulations once again! You now have demonstrated on how to verify HTML pages through URL from your running ZendFramework 2

FAQ

Why are my methods not being tested? You have to add a suffix Test in order for it to be included in running the tests.

Is “setup” class the same as construct? Kind of the only difference is “setup” is run for every test method.

Is it possible to make multiple assertions in one method? Yes, you can have as many assertions as you like, just make sure your methods are inside test suffixes or setup method
